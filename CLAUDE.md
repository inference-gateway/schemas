## What this repo is

The `inference-gateway/schemas` repo holds the three shared schemas the rest of the ecosystem (gateway, SDKs, docs, CLI, operator) consumes:

- `openapi.yaml` — Inference Gateway's HTTP API spec. **Hand-edited; source of truth.**
- `a2a/` — A2A (Agent-to-Agent) protocol. `a2a.proto` is the source of truth; `a2a-schema.json` / `.yaml` are **generated** from it.
- `mcp/` — Model Context Protocol schema. **Mirrored** from `modelcontextprotocol/modelcontextprotocol`; both the JSON and YAML are generated.

Downstream consumers regenerate from these files, so a schema change here ripples into `inference-gateway/sdks`, `inference-gateway/docs`, and others. See the `maintainer` skill / cross-repo checklist before shipping a breaking change.

## Commands

All tasks go through `Taskfile.yml`. `task --list` enumerates them; the load-bearing ones are:

| Task | What it does |
| --- | --- |
| `task openapi:lint` | Spectral lint of `openapi.yaml` (CI runs this on every push/PR). |
| `task openapi:format` | `prettier --write openapi.yaml` (single quotes, 2-space). |
| `task mcp-schema-download` | Fetches latest MCP JSON schema from upstream, regenerates the YAML. |
| `task a2a-schema-download` | Runs `buf generate` on `a2a/a2a.proto` → bundled JSON schema → post-processed `a2a-schema.{json,yaml}`. Requires Go + `buf` on PATH. |
| `task release:dry` | Previews the next semantic-release version + notes locally (`npx semantic-release --dry-run`); publishes nothing. |

`node scripts/check-reachable.js [path]` — sanity-checks that streaming-payload schemas (`CreateChatCompletionStreamResponse` and friends) are still reachable from operations in `openapi.yaml`. `oapi-codegen` drops unreachable schemas during code generation; this prevents regressions of issue #31.

The two schema-sync tasks are also exposed as `workflow_dispatch` GitHub Actions (`a2a-schema-sync.yml`, `mcp-schema-sync.yml`) that open a `chore: Sync …` PR automatically.

Releases are automated with **semantic-release** (config in `.releaserc.yaml`), triggered manually via the `workflow_dispatch` `Release` workflow (`release.yml`). It derives the version from Conventional Commits, updates `CHANGELOG.md`, tags, and creates a GitHub Release. Nothing is published to a package registry. See `RELEASING.md`.

## Generated files — do not hand-edit

- `a2a/a2a-schema.json`, `a2a/a2a-schema.yaml` — regenerated by `task a2a-schema-download`. To change A2A types, edit `a2a/a2a.proto` (or wait for upstream and re-sync), then rerun the task.
- `mcp/mcp-schema.json`, `mcp/mcp-schema.yaml` — mirrored from upstream. Don't patch locally; fix upstream or rerun `task mcp-schema-download`.

`openapi.yaml` **is** the source — edit it directly.

## A2A generation pipeline (why the scripts exist)

`task a2a-schema-download` is a four-stage pipeline; each script in `scripts/` handles one stage that `buf`/`protoc-gen-jsonschema` alone can't do:

1. **`buf generate`** — emits per-message `*.jsonschema.strict.bundle.json` files into `a2a/` using the `target=json-strict-bundle` plugin option.
2. **`scripts/process-bundle.js`** — merges all bundle files into a single `a2a-schema.{json,yaml}` under `definitions:`, then:
   - strips MkDocs `--8<-- [start:X]` / `[end:X]` snippet markers from descriptions (proto comments include them so docs can include slices verbatim);
   - strips namespace prefixes (`a2a.v1.`, `google.protobuf.`) from definition keys **and** `$ref` strings;
   - rewrites `#/$defs/` → `#/definitions/`;
   - deletes `patternProperties` (downstream codegen doesn't handle them);
   - removes the per-message bundle files.
3. **`scripts/parse-proto-required.js`** — parses `a2a.proto` for `[(google.api.field_behavior) = REQUIRED]` annotations and returns `{ MessageName: [camelCaseField, …] }`. JSON Schema's `required:` keyword isn't produced by `protoc-gen-jsonschema`, so we recover it ourselves.
4. **`scripts/add-required-fields.js`** — consumes that map and writes `required:` arrays back into the merged schema (both `.json` and `.yaml`).

If you touch any of these scripts, run `task a2a-schema-download` end-to-end and diff the output against `main` — they're load-bearing for SDK codegen.

## Conventions

- **Commits:** Conventional Commits, capitalized description (`chore: Sync MCP schema`, `feat(openapi): Add foo endpoint`). Semantic-release downstream depends on this.
- **YAML/Markdown style:** 2-space indent, single quotes, LF endings, final newline (`.editorconfig` + `.prettierrc`).
- **CODEOWNERS:** `a2a/*` is owned by `@inference-gateway/a2a` — expect that team on any A2A-touching PR.
- **CI:** the only check is `spectral lint openapi.yaml`. There's no test suite; the sync workflows themselves are the regression net for the generated schemas.
